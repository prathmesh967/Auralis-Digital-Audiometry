# Auralis - Digital Audiometry

This repository is now organized into a frontend/backend workspace structure.

## Structure

- `frontend/` — React + Vite application (PWA)
- `backend/` — Express API server
- `legacy_ui/` — archived original HTML pages

## Root commands

From the repository root:

- Install all dependencies:
  ```bash
  npm install
  ```
- Start the frontend in development:
  ```bash
  npm run dev
  ```
- Build the frontend production bundle:
  ```bash
  npm run build
  ```
- Start the backend server:
  ```bash
  npm run start
  ```

## Frontend package

The frontend package lives in `frontend/`.

- `frontend/package.json` contains frontend dependencies and scripts.
- `frontend/src/` contains the React application.
- `frontend/dist/` contains the build output.
- PWA features: Service worker for offline caching, web app manifest for installability.

## PWA Setup

The app is configured as a Progressive Web App (PWA):

- **Manifest**: `public/manifest.json` defines app metadata and icons.
- **Service Worker**: Auto-generated by `vite-plugin-pwa` for caching and offline support.
- **Icons**: Add `icon-192x192.png` and `icon-512x512.png` to `public/` for app icons.

To add icons:
1. Create 192x192 and 512x512 PNG icons.
2. Place them in `frontend/public/` as `icon-192x192.png` and `icon-512x512.png`.

## Deployment

### Frontend Deployment

The frontend is a static PWA that can be deployed to any static hosting service:

1. Build the app:
   ```bash
   cd frontend
   npm run build
   ```
2. Deploy the `dist/` folder to:
   - Vercel
   - Netlify
   - GitHub Pages
   - Firebase Hosting
   - Any static server

Example for Vercel:
```bash
npm install -g vercel
cd frontend
vercel --prod
```

### Backend Deployment

The backend is an Express server. Deploy to:
- Heroku
- Railway
- Render
- DigitalOcean App Platform

Set environment variables in your deployment platform.

## Backend package

The backend package lives in `backend/`.

- `backend/package.json` contains backend dependencies and scripts.
- `backend/index.js` is the Express server entrypoint.
- `backend/.env.example` shows environment variables.

## Backend environment variables

Copy `backend/.env.example` to `backend/.env` and set:

```env
MONGODB_URI=mongodb://127.0.0.1:27017/sonic-architect
JWT_SECRET=your-secret-key
PORT=4000
```

Then start the backend with:

```bash
npm run start
```

## App routes

- `/` – Splash page
- `/login` – Login screen
- `/dashboard` – Home dashboard
- `/hearing-test` – Hearing test module
- `/results` – Audiogram report
- `/speech-test` – Speech audio and noise meter
- `/settings` – Settings page
